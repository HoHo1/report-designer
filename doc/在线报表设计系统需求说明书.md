# 在线报表设计系统需求说明书

## 一、文档概述

### 1\.1 文档目的

本文档详细定义**基于 SpringBoot3\+Vue3 的在线报表设计系统**的功能需求、非功能需求、接口规范及业务流程，为系统设计、开发、测试、验收提供统一依据，确保项目团队精准实现核心目标：**类 Excel 在线报表设计、多类型动态数据集绑定、常规表格数据报表渲染**，不涉及复杂交叉报表。

### 1\.2 适用范围

本系统适用于企业 / 团队内部数据报表快速设计、生成、预览、导出与管理，覆盖报表设计者、报表使用者、系统管理员三类角色，核心支撑业务数据可视化、数据统计、报表共享等场景。

1. SpringBoot3 官方开发文档

2. Vue3 \+ Vite 官方开发文档

3. Excel 基础操作规范

## 二、系统总体说明

### 2\.1 系统定位

轻量级、高可用的**在线可视化报表设计与管理系统**，无需本地安装 Excel，通过浏览器即可完成报表模板设计、数据集动态绑定、数据实时渲染、报表导出与共享，降低报表制作门槛，支持多数据源灵活接入。

### 2\.2 技术架构

- **前端**：Vue3 \+ Vite \+ TypeScript \+ Element Plus \+ 类 Excel 报表设计组件（支持表格编辑、样式配置）

- **后端**：SpringBoot3 \+ MyBatis\-Plus \+ MySQL \+ Redis（缓存） \+ 定时任务（数据集刷新）

- **核心依赖**：

    1. 报表渲染：原生表格渲染引擎

    2. 文件处理：EasyExcel（静态文件解析、报表导出）

    3. 数据库连接：Druid（SQL 数据集连接池）

    4. 接口调用：RestTemplate（API 数据集请求）

- **部署方式**：前后端分离部署，支持 Docker 容器化

### 2\.3 核心设计原则

1. **易用性**：类 Excel 操作逻辑，零代码 / 低代码设计报表

2. **灵活性**：支持 API、SQL、静态文件三类数据集动态绑定

3. **稳定性**：数据渲染容错、异常捕获、连接超时保护

4. **扩展性**：支持后续新增数据集类型、报表样式扩展

5. **专一性**：仅实现**常规表格数据报表**，不开发交叉报表、聚合报表等复杂格式

### 2\.4 核心功能边界

✅ **支持**：

- 类 Excel 在线表格设计（单元格编辑、合并、样式、边框）

- API/SQL/ 静态文件数据集创建、管理、测试

- 数据集与报表单元格动态绑定

- 动态数据实时渲染、预览

- 报表导出（Excel/PDF）、复制、删除、共享

- 报表模板管理、权限控制

❌ **不支持**：

- 复杂交叉报表、多维聚合报表、图表报表

- 宏命令、Excel 高级函数（仅支持基础统计函数）

- 大数据量离线计算、超大型报表（单报表数据量限制 1 万行内）

## 三、角色与权限

### 3\.1 角色定义

|角色|职责|核心权限|
|---|---|---|
|系统管理员|系统配置、用户管理、数据集管理|用户增删改查、所有报表管理、全局配置、数据集审核|
|报表设计者|报表模板设计、数据集绑定、预览导出|个人报表管理、公共数据集使用、私有数据集创建|
|报表使用者|报表查看、数据刷新、导出打印|查看授权报表、刷新数据、导出报表、打印报表|

### 3\.2 权限控制

1. 数据权限：报表设计者仅能使用自己创建 / 管理员授权的数据集

2. 操作权限：报表使用者仅可查看 / 导出，不可编辑报表模板

3. 隔离权限：私有报表仅本人可见，公共报表全员可见

## 四、功能需求详述

本系统分为**前端报表设计器模块、数据集管理模块、报表管理模块、系统管理模块**四大核心模块。

---

### 4\.1 前端报表设计器（核心模块）

**目标**：提供类 Excel 在线编辑界面，支持常规表格设计、数据集绑定、数据实时渲染。

#### 4\.1\.1 基础表格编辑功能

1. **单元格操作**

    - 支持单元格文本输入、数字、日期格式编辑

    - 支持单元格合并 / 取消合并、跨行跨列设置

    - 支持单元格内容对齐（左 / 中 / 右、上 / 中 / 下）

    - 支持单元格换行、清空内容

2. **行列操作**

    - 支持插入 / 删除行、插入 / 删除列

    - 支持调整行高、列宽

    - 支持冻结表头（固定顶部 N 行）

    - 限制：最大支持 100 列、1 万行数据渲染

3. **样式配置**

    - 字体：大小、颜色、加粗、斜体、下划线

    - 背景：单元格背景色填充

    - 边框：全边框、内边框、外边框、边框颜色 / 粗细

    - 预设样式：提供表头、内容区默认样式模板

#### 4\.1\.2 数据集动态绑定（核心）

1. **数据集选择**

    - 设计器右侧展示当前用户可用的所有数据集（API/SQL/ 静态文件）

    - 支持数据集搜索、筛选类型、查看数据集字段

2. **字段绑定**

    - 选中目标单元格 / 列，选择数据集字段进行绑定

    - 支持**整列数据绑定**（常规表格核心：表头绑定字段名，内容行绑定数据集）

    - 支持**单个单元格静态数据 \+ 动态数据混合**

    - 绑定后自动显示字段占位符（如：\{userName\}）

3. **数据渲染配置**

    - 支持数据排序（升序 / 降序，绑定指定字段）

    - 支持数据过滤（简单条件：等于、大于、小于、包含）

    - 支持分页渲染（前端分页，默认每页 20 行）

    - 实时预览：绑定后一键渲染真实数据，查看效果

#### 4\.1\.3 报表预览与操作

1. 实时预览：渲染动态数据，展示最终报表效果

2. 预览模式：支持全屏预览、自适应屏幕

3. 撤销 / 重做：支持编辑操作撤销、重做

4. 自动保存：设计过程中每 30 秒自动保存草稿

5. **报表分页设置**分页方式：后端分页

6. 分页参数配置：可自定义每页显示条数（范围10\-100条，默认20条），支持设置分页显示样式（页码显示、上一页/下一页按钮、首页/尾页按钮）

7. 分页预览：设置完成后实时预览分页效果，支持快速切换页码查看不同页数据

8. 分页保存：分页设置与报表模板同步保存，导出报表时可选择导出全部数据或当前页数据

---

### 4\.2 数据集管理模块（核心模块）

**目标**：支持三类数据集创建、测试、管理、授权，为报表提供动态数据来源。

#### 4\.2\.1 公共能力（所有数据集通用）

1. **基础信息**

    - 数据集名称、描述、类型、创建人、创建时间

    - 数据集状态：草稿 / 可用 / 禁用

2. **操作能力**

    - 新增、编辑、删除、复制、测试连通性

    - 查看数据集字段（字段名、类型、示例值）

    - 支持数据集授权给指定用户 / 角色

#### 4\.2\.2 SQL 数据集

1. **配置项**

    - 数据库类型：MySQL、Oracle、PostgreSQL、SQL Server

    - 连接信息：主机、端口、库名、账号、密码

    - SQL 查询语句：支持 SELECT 查询语句（禁止 UPDATE/DELETE/ALTER 等操作）

    - 超时时间：默认 10 秒，防止慢查询

2. **校验规则**

    - 必须为查询语句，系统自动校验 SQL 安全性

    - 支持参数化查询（如：WHERE id = \#\{id\}）

3. **测试**：输入参数，执行 SQL，返回前 100 条示例数据

#### 4\.2\.3 API 数据集

1. **配置项**

    - 请求地址：HTTP/HTTPS 接口 URL

    - 请求方式：GET/POST

    - 请求头：支持自定义 Header（如 Token、Content\-Type）

    - 请求参数：Query 参数、Body 参数（JSON 格式）

    - 数据解析路径：JSONPath 配置（指定返回数据的数组节点）

2. **校验规则**

    - 接口必须返回标准 JSON 格式

    - 超时时间：默认 10 秒

3. **测试**：发送请求，解析并展示返回字段与示例数据

#### 4\.2\.4 静态文件数据集

1. **支持文件**：Excel（\.xlsx）、CSV（\.csv）

2. **配置项**

    - 上传文件、选择工作表

    - 自动解析表头与数据行

3. **限制**

    - 单文件大小≤20MB

    - 最大数据行≤1 万行

4. **测试**：解析文件，展示字段与前 100 条数据

---

### 4\.3 报表管理模块

**目标**：实现报表模板的全生命周期管理，支持预览、导出、共享、分类。

#### 4\.3\.1 报表模板管理

1. **报表创建**

    - 空白报表创建（进入设计器）

    - 模板复制创建（基于现有报表复制后修改）

2. **报表列表**

    - 展示：报表名称、创建人、类型、创建时间、更新时间、权限范围

    - 操作：编辑、预览、复制、导出、删除、设置权限

3. **报表分类**

    - 支持自定义报表分类文件夹（如：销售报表、财务报表）

    - 支持报表移动、重命名、搜索筛选

#### 4\.3\.2 报表数据操作

1. **数据刷新**：重新请求数据集，渲染最新数据

2. **报表导出**

    - 导出格式：Excel（\.xlsx）、PDF

    - 导出内容：带样式 \+ 真实数据的完整报表

3. **报表打印**：浏览器在线打印预览、直接打印

4. **报表分享**：生成分享链接 / 二维码，设置有效期与查看权限

#### 4\.3\.3 报表版本管理

- 支持自动保存历史版本（最多保存 10 个）

- 支持版本回滚、版本对比、删除历史版本

---

### 4\.4 系统管理模块

**目标**：保障系统稳定运行，提供系统配置、日志管理能力。

**参数配置**

- 全局配置：报表最大行数、文件上传大小、接口超时时间

- 样式配置：默认报表字体、颜色、边框

1. **日志管理**

    - 操作日志：报表编辑、导出、删除记录

    - 数据集日志：SQL 执行、API 请求、文件解析日志

    - 错误日志：数据渲染失败、连接失败日志

2. **缓存管理**

    - 数据集结果缓存（默认 5 分钟，可配置）

    - 手动清理缓存、刷新数据

## 五、非功能需求

### 5\.1 性能需求

1. **响应时间**

    - 页面加载：≤2 秒

    - 报表设计操作：无卡顿，实时响应

    - 数据渲染：≤3 秒（1 万行内）

    - 数据集测试：≤5 秒

2. **并发支持**

    - 支持 50 人同时在线编辑

    - 支持 100 人同时查看 / 导出报表

3. **数据限制**

    - 单报表渲染数据≤10000 行

    - 静态文件上传≤20MB

    - API/SQL 请求超时≤10 秒

### 5\.2 易用性需求

1. 类 Excel 操作逻辑，无需培训即可上手

2. 操作提示清晰，错误信息明确，支持一键修复

3. 支持快捷键操作（复制、粘贴、撤销、保存）

4. 移动端适配：支持手机 / 平板查看报表，不支持编辑

### 5\.3 稳定性需求

1. 系统全年可用率≥99\.5%

2. 自动捕获异常，不崩溃、不丢失编辑数据

3. 数据库连接失败、接口超时、文件解析错误均有友好提示

4. 支持定时备份报表模板与数据集配置

### 5\.4 安全性需求

1. **身份认证**：基于 Token 登录，密码加密存储

2. **SQL 防注入**：SQL 数据集强制参数化，禁止危险语句

3. **接口安全**：API 请求鉴权，防止越权调用

4. **文件安全**：静态文件查杀，禁止上传可执行文件

5. **操作审计**：所有敏感操作记录日志

### 5\.5 兼容性需求

1. 浏览器：Chrome、Edge、Firefox 最新版（优先）

2. 分辨率：1920×1080 及以上自适应

3. 后端：JDK 17\+，MySQL 8\.0\+

## 六、业务流程

### 6\.1 报表设计完整流程

1. 报表设计者创建 / 管理数据集（API/SQL/ 静态文件）→ 测试连通性

2. 新建空白报表 → 进入类 Excel 设计器

3. 编辑表格基础样式、表头、固定内容

4. 选择数据集 → 绑定字段到表格列 / 单元格

5. 配置数据过滤、排序 → 实时预览渲染效果

6. 保存报表模板 → 发布为可用报表

7. 报表使用者查看、刷新、导出报表

### 6\.2 数据集使用流程

1. 管理员 / 设计者创建数据集 → 配置参数

2. 测试数据集连通性与数据解析

3. 保存数据集

4. 设计者在报表设计器中选择绑定

5. 系统自动拉取数据 → 渲染报表

## 八、接口规范（前后端交互）

### 8\.1 基础规范

- 请求方式：GET/POST/PUT/DELETE

- 数据格式：JSON

- 统一返回值：

```json
{
  "code": 200,
  "msg": "成功",
  "data": {}
}
```

### 8\.2 核心接口

1. **数据集接口**

    - 创建 / 编辑 / 删除 / 查询数据集

    - 测试数据集连通性

    - 获取数据集字段列表

2. **报表设计接口**

    - 保存 / 加载报表模板

    - 渲染报表数据（根据绑定关系查询数据集）

    - 导出报表（Excel/PDF）

3. **用户接口**

    - 登录 / 注销 / 用户信息

4. **系统接口**

    - 获取全局配置

    - 操作日志查询

## 九、验收标准

1. **功能验收**

    - 类 Excel 设计器所有基础功能可用

    - 三类数据集均可正常创建、测试、绑定

    - 动态数据绑定后可实时渲染、预览

    - 报表导出（Excel/PDF）格式正常、数据完整

2. **性能验收**

    - 1 万行数据渲染≤3 秒

    - 页面响应无卡顿

3. **兼容性验收**

    - 主流浏览器正常运行

4. **安全性验收**

    - 越权操作被拦截

    - SQL 注入、非法文件上传被拦截

## 十、附录

### 10\.1 术语说明

1. **常规表格报表**：标准二维表格，一行一条数据，一列一个字段，无合并行 / 列的复杂结构

2. **数据集**：报表的数据来源，分为 API、SQL、静态文件三类

3. **动态绑定**：将数据集字段与报表单元格关联，自动填充数据

4. **数据渲染**：根据绑定关系，将数据集真实数据填充到报表中
